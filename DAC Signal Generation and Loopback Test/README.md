# FPGA ADC-DAC Loopback Test with Sawtooth Waveform Generation

## Overview

This project implements a complete **ADC-DAC Loopback Test System** on an FPGA-based control board.

The design generates a digital sawtooth waveform inside the FPGA, outputs it through one DAC channel, physically routes the resulting analog signal to an ADC channel, samples the signal back into the FPGA, and finally reconstructs it through a second DAC channel.

The objective is to verify the complete signal chain:

```text
FPGA вҶ’ DAC3 вҶ’ Analog Signal вҶ’ ADC1 вҶ’ FPGA вҶ’ DAC1
```

This experiment forms the foundation for future Digital Signal Processing (DSP) applications such as:

* Digital Filtering
* FIR Filters
* IIR Filters
* Signal Conditioning
* Audio Processing
* Control Systems
* Power Electronics Signal Acquisition

# Sawtooth Waveform Generation

DAC3 is used as a waveform generator.

A 12-bit counter continuously increments:

```text
0 вҶ’ 1 вҶ’ 2 вҶ’ ... вҶ’ 4095 вҶ’ 0
```

creating a full-scale digital ramp.

Since DAC3 is a 12-bit DAC:

```text
Resolution = 2^12 = 4096 Levels
```

The DAC converts this ramp into an analog sawtooth waveform.

---

# ADC and DAC Data Formats

Understanding the data representation is critical for this experiment.

---

## DAC Format

The DACs on the board use:

```text
12-bit Offset Binary
```

Range:

```text
0000 0000 0000  = Minimum Output

1000 0000 0000  = Mid-scale

1111 1111 1111  = Maximum Output
```

Numerical Range:

```text
0 to 4095
```

---

## ADC Format

The ADC uses:

```text
10-bit Two's Complement
```

Range:

```text
1000000000 = -512

0000000000 = 0

0111111111 = +511
```

Numerical Range:

```text
-512 to +511
```

---

# Why Format Conversion Is Necessary

The ADC and DAC use different numerical representations.

| Device | Format                  |
| ------ | ----------------------- |
| ADC    | 10-bit Two's Complement |
| DAC    | 12-bit Offset Binary    |

Directly sending ADC output to the DAC would produce an incorrect waveform.

Therefore conversion is required.

---

## Two's Complement вҶ’ Offset Binary

The conversion is performed using:

```verilog
{~adc_data1[9], adc_data1[8:0]}
```

The MSB is inverted.

Example:

```text
Two's Complement:

1000000000 = -512

After MSB inversion:

0000000000
```

which corresponds to the minimum DAC value.

Similarly:

```text
0111111111 = +511

becomes

1111111111
```

which corresponds to the maximum DAC value.

---

## 10-bit вҶ’ 12-bit Scaling

After conversion:

```verilog
{~adc_data1[9], adc_data1[8:0], 2'b00}
```

two zeros are appended.

This performs:

```text
Value Г— 4
```

allowing the 10-bit ADC range to occupy the full 12-bit DAC range.

Without this scaling:

* DAC dynamic range would be underutilized
* Signal amplitude would be reduced
* Effective resolution would decrease

---

# Q-Format Representation

The design does not explicitly use fractional fixed-point arithmetic.

Instead it uses integer sample representations.

However the data can be interpreted as:

### ADC

```text
Q9.0
```

Meaning:

* 1 Sign Bit
* 9 Integer Bits
* 0 Fractional Bits

Range:

```text
-512 to +511
```

---

### DAC

```text
UQ12.0
```

Meaning:

* Unsigned
* 12 Integer Bits
* 0 Fractional Bits

Range:

```text
0 to 4095
```

# Signal Integrity Analysis

This experiment allows investigation of several practical effects.

---

## Quantization Error

ADC resolution:

```text
10 bits = 1024 Levels
```

DAC resolution:

```text
12 bits = 4096 Levels
```

The original DAC3 signal contains more levels than the ADC can represent.

Therefore quantization occurs.

Expected observation:

```text
DAC1 waveform appears slightly stair-stepped.
```

---

## Amplitude Accuracy

Compare:

* Peak value of DAC3
* Peak value of DAC1

Differences indicate:

* Gain error
* Reference voltage mismatch

---

## Offset Error

Compare waveform centers.

Differences indicate:

```text
ADC Offset Error
or
DAC Offset Error
```

---

## Noise

Small fluctuations in DAC1 but not DAC3 indicate:

```text
ADC Noise
DAC Noise
Board Noise
```

---

## End-to-End Latency

DAC1 output will be delayed relative to DAC3.

The delay includes:

```text
ADC Sampling
FPGA Processing
DAC Reconstruction
```

This can be measured using an oscilloscope.
